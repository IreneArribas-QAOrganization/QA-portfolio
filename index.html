Projects

"Vida Verde" Website – QA Manual y Automation

I performed Quality Assurance for a pre-existing web application, working within a cross-functional team under Scrum agile methodologies. Project management was conducted in weekly sprints using Jira where I actively participated in Sprint Planning, Dailies, and Sprint Reviews at the end of each sprint.

I designed test cases for User Stories following the BDD methodology, based on defined acceptance criteria. I authored scenarios in Gherkin to ensure tests were readable and understandable for the entire team, managing them with AssertThat to maintain full traceability between requirements, tests, and results within Jira. Execution was performed per User Story, recording results across both web and mobile environments.

During each sprint, I conducted functional testing, followed by exploratory testing at the end of the cycle to identify defects not covered by predefined cases. Detected bugs were reported in Jira and linked to their corresponding User Stories to ensure end-to-end traceability between requirements, tests, and defects.

At the close of each sprint, I prepared the Release in Jira and documented delivery-related information in Confluence, including comprehensive test plans, execution results, and the overall product quality status.

Following the release, I identified the application’s core business flows and automated regression testing using Playwright and Pytest with Python. I integrated these tests into a CI/CD pipeline using GitHub Actions, configured to automatically run the regression suite on weekly basis, aligned with each sprint’s closure.
